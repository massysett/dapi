NAME

dapi - print a series of dates

SYNOPSIS

dapi [options] DATE_RANGE...

DESCRIPTION

Ranges
dapi prints a series of dates on standard output. You specify one or more ranges of dates. Examples:

dapi ’this month’dapi ’next month’dapi ’jan’dapi ’this week’dapi ’+1 week’

Often ranges include multiple words, so you have to quote them so that your shell presents them to dapi as a single command line argument. Also, you might specify a range that leads with a minus sign. Before doing that, use a double dash to indicate that you are not entering options anymore, otherwise dapi will get confused and think you are entering an option:

dapi -- ’-2 year’

You can specify exact dates for your range:

dapi ’2013-03-01 - 2013-04-01’

This is a single range, so you must quote it as it is a single command-line argument. The second date in the range is the date that is one day past the last day you want included in the range. Therefore, the above example prints every day in March 2013; it does not include April 1.

Ranges can also be backward:

dapi ’2013-03-31 - 2013-02-28’

prints every day in March 2013, but in reverse order. In situations like that it can be handy to specify that you want to use the last day of the month. Use the letter "l" for this. The next example prints the same dates as the one above:

dapi ’2013-03-l - 2013-02-l’

You can specify more than one range and all the dates will be printed. This prints every day in 2013 and 2014:

dapi 2013 2014

Filters
If you don’t want to see all the dates in the ranges that you specified, you can filter the days. You specify a predicate--that is, a function that will be applied to each date in the ranges. The date is included in the result only if the predicate returns True. For instance, the --day option specifies a predicate which examines the day of the month. This prints the first day of each month in 2013:

dapi --day ’==’ 1 2013

Prints the first and second day of each month in 2013:

dapi --day ’<=’ 2 2013

As with range specifiers, you can use the letter "l" with the --day option to specify the last day of the month. Prints the last day of each month in 2013:

dapi --day ’==’ l 2013

Another predicate option is --weekday, which specifies a day of the week. Prints every Thursday in this month:

dapi --weekday ’==’ thu ’this month’

If you have more than one predicate option, you must join them together using operators. The operator options are --and, --or, and --not. By default, dapi uses reverse polish notation, which (imprecisely) means that operators are specified after the predicates they affect. For example, this prints every date in 2013 that is both a Sunday and is in the first 15 days of the month:

dapi --weekday ’==’ sun --day ’<=’ 15 --and 2013

OPTIONS

The order of the dapi command line is significant. Most options only affect other options that are later on the command line. This is similar, for better or for worse, to find(1).

Options to build predicates
Many of these options take a comparer. Here are the valid comparers. You may need to quote many of them, otherwise your shell will interpret them to mean something else (redirection, for example.)

--date, -t COMPARER DATE_SPEC

Date falls within the specified range. DATE_SPEC may be in YYYY-MM-DD format, or it can be a relative date, like tue.

--weekday, -w COMPARER DAY_OF_WEEK

Day of week falls in the specified range. The order of days is affected by the --week-start option. DAY_OF_WEEK is a three-letter lowercase abbreviation for the day of the week.

--day, -d COMPARER DAY_OR_LAST

The day number falls in the given range. DAY_OR_LAST either is digits specifying the day, or the letter "l" meaning the last day of the month.

--year, -y COMPARER YEAR

The year number falls in the given range. YEAR is digits.

--month, -m COMPARER MONTH

The month falls in the given range. Currently the ordering is set so that January is always the first month. MONTH is a three-letter abbreviation for a month.

--ends, -e

The first or the last day in the entire list of dates. For example

dapi --ends ’this year’

shows the first and last days of the year.

--first, -f

The first day in the list of dates.

--last, -l

The last day in the list of dates.

--count, -c DIGITS

Selects every nth day. For example, to see your paydays if you get paid every 2 weeks staring on January 10, 2013:

dapi --count 14 ’2013-01-10 - 2014-01-01’

--and, -a, --or, -o, --not, -n

The operators. By default dapi uses reverse polish notation; for infix notation, use the --infix option. --and and --or are binary; --not is unary. For instance, to show the first day of every month this year that is not in June or July:

dapi --day ’==’ 1 --month ’==’ jun --month ’==’ jul \
     --or --not --and ’this year’

Another way to do this:

dapi --day ’==’ 1 --month ’/=’ jun --month ’/=’ jul \
     --and --and ’this year’

--open, -(, --close, -)

Open and close parentheses. You can only use these when using the
--infix
option, as reverse polish notation expressions cannot use parentheses.

Options affecting dates
--current DATE_SPEC, -C DATE_SPEC

Sets the current date. When you specify a date or a range that is relative to a day, such as in the command dapi ’this month’, dapi needs to start with a date to figure out when "this month" is. By default, dapi uses today. To specify a different date, use this option.

--week-start, -W DAY_OF_WEEK

The first day of the week is the specified day.

--format FORMAT, -f FORMAT

Formats dates. This uses the same formatting strings that strftime(3) uses, so look at that man page for your choices here. A newline is appended to the end of each date automatically.

--base1

By default, centuries, decades, and millennia begin in years that are evenly divisible by 10. However, since the first year was year 1 rather than year 0, the first century was years 1-100, not 0-99. This option makes dapi conform to this. The converse is --base0, which is the default. For example, compare the output of

dapi --base1 --day ’==’ 1 --month ’==’ jan --and ’this century’

with

dapi --base0 --day ’==’ 1 --month ’==’ jan --and ’this century’

Options affecting expressions
--infix, -I

Use infix operators. By default dapi uses reverse polish notation operators.

--rpn, -R

Use reverse polish notation operators

--show-expression, -S

Show the expression used to filter dates. Useful if you enter a complex expression and you want to make sure it is doing what you think it is.

--verbose-filter, -V

Be verbose when filtering the days. This shows you which days are included in the final list of days, which days are rejected, and why. By default this is colorful, unless your terminal does not support colors or unless standard output is not a terminal. To suppress colors all the time, preface your dapi command with TERM=dumb.

--color-to-file, -T

Use colors even when standard output is not a terminal. Affects only the --verbose-filter option.
